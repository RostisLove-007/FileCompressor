Detailed Description

Высокоуровневый класс сжатия и распаковки файлов в форматы CZIP/CRAR.

Compressor инкапсулирует выбор алгоритма по CompressMethod, чанковую обработку больших файлов, отчёт о прогрессе и статусе через callback-и, проверку целостности через CRC32 и поддержку возобновления прерванного сжатия с использованием контрольных точек (см. archive.hpp).

Запрашивает отмену текущей выполняющейся операции.

Отмена не происходит мгновенно: она применяется на границе обработки очередного чанка, при этом для compress() будет сохранена контрольная точка, позволяющая позже продолжить операцию.

Сжимает файл в архив CZIP/CRAR с потоковой чанковой обработкой и поддержкой возобновления после отмены.

Последовательно: определяет тип исходного файла, при необходимости загружает контрольную точку и продолжает с последнего обработанного чанка, открывает (или дооткрывает) выходной архив, читает исходный файл по частям размером CHUNK_SIZE, сжимает каждый чанк, накапливает CRC32, записывает чанк в архив и обновляет контрольную точку. После обработки всех чанков финализирует заголовок архива (итоговый размер, CRC32, число чанков) и удаляет файл контрольной точки.

Parameters
input_pathПуть к исходному файлу, который требуется сжать.
output_pathПуть к создаваемому (или продолжаемому) файлу архива.
optsПараметры сжатия: формат, метод, размер окна, флаг возобновления.
Exceptions
std::runtime_errorесли исходный файл не найден, пуст, не удалось открыть/создать выходной файл, либо операция была отменена через cancel() (в этом случае перед выбросом исключения сохраняется контрольная точка).

void Compressor::decompress ( const std::string & input_path,
const std::string & output_path )
inline

Восстанавливает исходный файл из архива CZIP/CRAR (чанковая распаковка).

Читает архив целиком, разбирает заголовок и количество чанков, затем для каждого чанка распаковывает данные выбранным методом (либо копирует их «как есть», если чанк был сохранён без сжатия), накапливает CRC32 восстановленных данных и записывает их в выходной файл. После обработки всех чанков проверяет совпадение итогового размера и контрольной суммы с теми, что записаны в заголовке архива.

Parameters
input_pathПуть к файлу архива, который требуется распаковать.
output_pathПуть к создаваемому восстановленному файлу.
Exceptions
std::runtime_errorесли архив повреждён или имеет некорректную структуру, операция была отменена через cancel(), какой-либо чанк усечён или не прошёл распаковку, итоговый размер не совпадает с ожидаемым, либо контрольная сумма CRC32 не совпадает с указанной в заголовке архива.
